# AIccountant — правила проекта

AI-бухгалтер для домохозяйства: чтение финансовых документов, сверка,
аномалии, прогноз. Дизайн и дорожная карта:
`docs/superpowers/specs/2026-07-02-aiccountant-design.md` — сверяться с ней
перед крупными изменениями; решения из раздела «Принятые развилки»
не пересматривать молча.

## Структура репозитория

- `backend/` — FastAPI, модульный монолит (Python 3.12+, uv)
- `frontend/` — React 18 + TypeScript + Vite (pnpm), Mantine, PWA
- `infra/` — docker-compose, Caddy, деплой
- `docs/` — спеки, планы, ADR

## Архитектурные правила

- Модули backend: `identity`, `ledger`, `recurring`, `imports`, `ai`,
  `analytics`, `notifications`. Общение — только через сервисные интерфейсы;
  в таблицы чужого модуля не ходить (контролируется import-linter).
- Все доменные таблицы несут `workspace_id`; каждый запрос фильтрует по нему
  в repository-слое. Утечка данных между workspace — критический баг.
- Деньги — `NUMERIC(20, 4)` в БД и `Decimal` в коде. `float` для денег
  запрещён везде, включая тесты; на фронте суммы передаются строками.
- LLM/эмбеддинги/распознавание речи — только через интерфейсы модуля `ai`
  (`LLMClient`, `EmbeddingClient`, `Transcriber`). Вендорские SDK
  не импортировать за пределами реализаций этих интерфейсов.
- Фоновая и периодическая работа — Celery/beat. Доменная логика не живёт
  во внешних инструментах.
- Схема БД меняется только через миграции Alembic.

## Тулинг и качество

- Python: uv, ruff (линт и формат), mypy (strict), pytest + pytest-asyncio,
  testcontainers для интеграционных тестов.
- TypeScript: pnpm, eslint + prettier, vitest.
- Тесты обязательны для доменной логики и краевых случаев. LLM-фичи —
  контрактные тесты на записанных ответах; категоризация — eval-набор
  с метрикой точности.
- Перед «готово» — линт, типы и тесты прогнаны, результат подтверждён.

## Процесс

- Этап дорожной карты закрыт, только когда задеплоен на демо-URL и описан
  в README. Следующий этап раньше не начинается.
- В логи не писать PII и суммы — только идентификаторы.
